# 文档分类与目录结构规范 (V1.1)

## 1. 目的

为确保所有知识文档都能被精准、一致地归入知识库的目录体系中，避免分类混淆，提高信息检索效率，特制定本规范。

## 2. 文档归属判断流程（首要原则）

当创建一份新文档时，请按以下顺序提问，以确定其一级目录：

1. **是关于IT基础设施的技术细节吗？** -> `10_IT基础设施/`
2. **是关于某个业务软件系统的吗？** -> `20_系统与应用/`
3. **是一个标准化的、重复性的操作步骤吗？** -> `30_标准作业程序/`
4. **是属于某个有起止时间的IT项目吗？** -> `40_项目文档/`
5. **是用于团队培训或经验分享吗？** -> `50_知识分享/`
6. **涉及外部供应商的选型、报价或合同吗？** -> `60_采购与解决方案/`
7. **关于团队成员的招聘、成长或管理吗？** -> `70_团队建设与管理/`
8. **是一个软件安装包或驱动的下载索引吗？** -> `80_软件库与驱动/`

**核心原则：** 一份文档只归属于一个最主要的分类。

## 3. 目录结构规范与扩展原则

目录结构应遵循“从通用到具体”的逻辑下钻原则，按需创建。

### 3.1. `10_IT基础设施/`

* **二级 (按技术领域):** `/网络/`, `/服务器与虚拟化/`, `/信息安全/`
* **多级示例:** `/服务器与虚拟化/VMware-vSphere/`

### 3.2. `20_系统与应用/`

* **二级 (按系统名称):** `/ERP系统/`, `/CRM系统/`, `/OA系统/`
* **多级示例:** `/ERP系统/功能模块/供应链模块/`

### 3.3. `30_标准作业程序/`

* **二级:** 通常不设二级目录，或按场景分为 `/用户支持类/`, `/运维类/`。

### 3.4. `40_项目文档/`

* **二级 (按项目):** `/PROJ-CRM2025/`, `/PROJ-OA升级2026/`
* **多级示例:** `/PROJ-CRM2025/03_设计文档/架构设计/`

### 3.5. `50_知识分享/`

* **二级 (按技术或业务主题):** `/数据库专题/`, `/网络安全/`, `/项目管理实践/`

### 3.6. `60_采购与解决方案/`

* **二级 (按采购标的):** `/硬件采购/`, `/软件采购/`, `/服务采购/`
* **多级示例:** `/硬件采购/2025_服务器升级/戴尔/`

### 3.7. `70_团队建设与管理/`

* **二级 (按管理职能):** `/招聘与面试/`, `/培训与成长/`, `/绩效管理/`

### 3.8. `80_软件库与驱动/`

* **二级 (按类型):** `/公共软件/`, `/专用软件/`, `/硬件驱动/`
