# H.04-DMZ服务发布-SOP (深信服NGAF)

> **标签**: `#SOP` `#DMZ` `#服务发布` `#DNAT` `#深信服`
> **版本**: 1.0
> **状态**: 草稿

---

## 1. 概述

本SOP (Standard Operating Procedure) 提供了在**深信服下一代防火墙 (NGAF)** 上，将部署在DMZ区域的内部服务安全地发布到互联网的标准流程。

**目标**: 在提供外部访问能力的同时，严格控制访问权限，最大限度地保护内部网络安全。

## 3. 步骤一: 准备工作 (对象定义)

在配置策略之前，首先要清晰地定义“谁”、“访问哪里”、“用什么服务”。

1. **登录深信服NGAF的Web管理界面**。
2. **确认信息**:
    - **内部服务器IP**: 位于DMZ区域的服务器的私网IP (e.g., `172.16.99.10`)。
    - **对外服务端口**: 希望向公网开放的端口 (e.g., `TCP/443`)。
    - **公网IP**: 用于接收外部访问的防火墙公网IP (e.g., `x.x.x.x`)。
3. **创建地址对象**:
    - 导航至 `对象` -> `IP/组` -> `新建`。
    - 创建一个地址对象，命名为 `[服务名]_Private_IP` (e.g., `WebApp01_Private_IP`)，值为 `172.16.99.10`。
    - 确认已存在代表公网IP的地址对象，或新建一个，命名为 `[服务名]_Public_IP`。
4. **创建服务对象**:
    - 导航至 `对象` -> `服务` -> `新建`。
    - 创建一个服务对象，命名为 `[服务名]_Service` (e.g., `WebApp01_HTTPS`)，协议为`TCP`，目的端口为 `443`。

---

## 4. 步骤二: 配置端口发布 (DNAT)

这是实现地址映射的关键步骤。

1. 导航至 `防火墙` -> `端口发布` -> `新建规则`。
2. **规则名称**: 填写一个清晰的名称，如 `Publish_WebApp01_HTTPS`。
3. **发布IP**: 从下拉列表中选择代表**公网IP**的地址对象 (`[服务名]_Public_IP`)。
4. **发布端口**: 从下拉列表中选择刚刚创建的**服务对象** (`[服务名]_Service`)。
5. **内部服务器IP**: 从下拉列表中选择代表**内部服务器IP**的地址对象 (`[服务名]_Private_IP`)。
6. **内部端口**:
    - 如果内外端口一致 (e.g., 都是443)，则留空或选择与发布端口一致。
    - 如果需要做端口转换 (e.g., 外网访问8443，内部是443)，则在此处指定内部端口 `443`。
7. **源IP限制**:
    - **强烈建议**: 如果服务只对特定伙伴开放，请在此处选择代表伙伴公网IP的地址对象，实现源地址过滤。
    - 如果服务对所有人开放，则保持 `any`。
8. 点击 `确定` 保存。

---

## 5. 步骤三: 审查和优化安全策略

创建端口发布规则后，系统会自动生成一条对应的安全策略。我们需要检查并根据需要进行优化。

1. 导航至 `防火墙` -> `安全策略` -> `IPv4策略`。
2. 找到系统自动生成的那条策略（通常与端口发布规则同名）。
3. **检查策略要素**:
    - **源安全域**: `untrust`
    - **目的安全域**: `dmz`
    - **源IP/组**: 确认与端口发布中设置的源IP限制一致。
    - **目的IP/组**: 应为**公网IP对象** (`[服务名]_Public_IP`)。
    - **服务**: 应为创建的**服务对象** (`[服务名]_Service`)。
    - **动作**: `允许`。
4. **开启日志**:
    - 编辑该策略，在 `策略选项` 中，确保 `记录日志` 已被勾选。这对于后续排错和审计至关重要。
5. **应用安全配置文件 (可选但推荐)**:
    - 如果您购买了IPS、WAF等功能模块，可以在 `防护配置` 中为这条策略启用相应的防护配置文件，增加应用层安全。
6. 保存策略。

---

## 6. 验证

1. **外部访问**:
    - 使用**外部网络** (如手机热点) 的设备，通过浏览器访问 `https://[公网IP]` 或 `https://[公网IP]:[发布端口]`。
    - 确认服务可以正常访问。
2. **日志检查**:
    - 导航至 `监控中心` -> `日志中心` -> `流量日志`。
    - 使用过滤器，筛选目的IP为公网IP、目的端口为发布端口的流量。
    - 确认可以看到来自外部访问的、动作为“允许”的日志记录。
