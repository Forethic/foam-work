# CM站点网络重置 - 验收方案 (UAT Plan)

> **项目**: [[../index.md|Sub-02-Network_Upgrade]]
> **执行项**: `EXEC-202512-CM_Network_Reset`

---

## 1. 验收概述

本验收方案旨在提供一套清晰、可量化的标准，用于验证纯铭(CM)站点的网络重置工作是否完全符合项目需求。验收过程需由我方项目负责人与供应商技术工程师共同在场完成。

---

## 2. 验收检查清单 (Checklist)

### 2.1. 过程文档与准备工作验收

| 验收项 | 验收标准与测试用例 | 预期结果 | 验收结果 (Pass/Fail) |
| :--- | :--- | :--- | :--- |
| **静态IP设备统计** | 1. 供应商是否已根据要求，填写完成了 `[[A.01.2-静态IP设备表-CM.md]]` 中的所有统计信息。<br>2. 我方负责人确认该表格覆盖了所有已知的静态IP设备。 | `A.01.2`文档中的统计信息完整、准确。 | |

### 2.2. TP-LINK核心交换机配置验收

| 验收项 | 验收标准与测试用例 | 预期结果 | 验收结果 (Pass/Fail) |
| :--- | :--- | :--- | :--- |
| **VLAN创建** | 1. 登录核心交换机，执行 `show vlan`。<br>2. 核对VLAN ID和名称是否与 `[[A.01-IP与VLAN分配-总表]]` 中CM站点的规划完全一致。 | 所有已规划VLAN均已创建，名称正确。 | |
| **SVI接口IP** | 1. 执行 `show running-config`。<br>2. 核对每个 `interface vlan` 的IP地址和子网掩码是否与总表规划完全一致。 | 所有SVI的IP地址和掩码正确无误。 | |
| **ACL对象组** | 1. 执行 `show running-config`。<br>2. 检查所有 `object-group` 的定义，是否与 `[[E.05.06-VLAN间访问控制(ACL)配置SOP]]` 中的定义完全一致。 | 所有对象组均已创建，且包含的IP/服务正确。 | |
| **ACL规则** | 1. 执行 `show access-lists 100`。<br>2. 逐条核对ACL规则，确保顺序、动作、源/目地址、服务与SOP中的定义完全一致。 | ACL 100的规则与SOP完全一致。 | |
| **ACL应用** | 1. 执行 `show running-config`。<br>2. 检查**所有**应受控的VLAN接口（用户、服务、访客等），确认接口下均已配置 `ip access-group 100 in` 命令。 | 所有目标VLAN接口均已正确应用ACL。 | |
| **功能测试** | 1. 按照ACL SOP中的“验证”章节，从不同VLAN的终端发起ping或服务访问测试。<br>2. 例如：从用户区ping管理区IP应失败；从IT运维区ping管理区IP应成功。 | 所有访问行为均与ACL策略矩阵的预期一致。 | |

### 2.2. 深信服设备配置验收

| 验收项 | 验收标准与测试用例 | 预期结果 | 验收结果 (Pass/Fail) |
| :--- | :--- | :--- | :--- |
| **防火墙接口与路由** | 1. 登录防火墙Web界面。<br>2. 检查`网络` -> `接口`，确认管理口、LAN/WAN口IP地址正确。<br>3. 检查`网络` -> `路由`，确认默认路由和内部回程路由正确。 | 接口IP和路由配置符合SOP要求。 | |
| **防火墙NAT与策略** | 1. 检查`防火墙` -> `NAT策略`，确认SNAT策略配置正确。<br>2. 检查`防火墙` -> `安全策略`，确认基础出站策略配置正确。 | 内网用户可通过防火墙正常访问互联网。 | |
| **AC与AD对接** | 1. 登录AC设备Web界面。<br>2. 检查`认证管理` -> `认证服务器` -> `AD/LDAP服务器`。<br>3. 点击“测试连接”。 | 连接状态显示成功。 | |
| **AC RADIUS配置** | 1. 检查`认证管理` -> `认证方式` -> `Radius认证`。<br>2. 确认RADIUS服务已启用，且无线AP/控制器已添加为客户端。 | RADIUS服务配置正确。 | |
| **IPsec VPN状态** | 1. 登录防火墙Web界面。<br>2. 检查`VPN` -> `IPsec` -> `IPsec`。<br>3. 确认与HZ站点的VPN隧道状态为“已连接”。 | VPN隧道已成功建立。 | |
| **功能测试** | 1. **VPN测试**: 从CM站点的用户PC `ping` HZ站点的服务器IP，应能ping通。<br>2. **WIFI测试**: 使用测试账户连接`Staff_WIFI`和`Guest_WIFI`，确认认证流程和网络访问符合预期。 | 跨站业务和WIFI认证功能正常。 | |
