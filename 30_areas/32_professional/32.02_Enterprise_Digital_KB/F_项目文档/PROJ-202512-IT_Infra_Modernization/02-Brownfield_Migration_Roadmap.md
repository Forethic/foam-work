## 1. 概述与最终策略

本文档是针对您当前IT环境及**“独立管理，按需协作”**的最终业务需求，制定的最终版、分阶段行动路线图。

**最终架构**: **多域森林**。我们将构建一个`corp.local`林，并在其中创建`huazhi.corp.local`和`chunming.corp.local`两个独立的子域，以实现管理和策略的彻底分离。

**最终策略：分期执行，先内后外，风险隔离**

- **第一阶段 (准备期 - 未来3周)**: 利用新设备到货前的窗口期，完成所有不依赖新设备的**内部准备工作**。
- **第二阶段 (实施期 - 设备到货后)**: 执行最核心的、风险最高的**网络割接**操作，将网络核心切换到新防火墙，并启用新的服务。

---

## 2. 第一阶段：准备期 (防火墙到货前)

- **目标**: 完成所有内部服务的部署和配置，使之处于“待命”状态。**本阶段所有操作均不影响现有网络运行**。

| 步骤 | 任务描述 | 影响范围 | 实施指南 (SOP) |
| :--: | :--- | :--- | :--- |
| **1** | **创建新VLAN** | 在**所有交换机**上，创建所有规划的新VLAN。 | 无 | [[A.02-VLAN与端口配置-SOP.md]] |
| **2** | **临时配置VLAN网关** | **只在**现有的核心交换机上，为所有新VLAN临时配置三层网关接口。 | 无 | [[A.02-VLAN与端口配置-SOP.md]] |
| **3** | **在ESXi中创建端口组** | 在ESXi-B上为**新的服务器VLAN (1010)** 创建一个新的虚拟端口组。 | 无 | - |
| **4** | **部署AD林与子域** | 在ESXi-B上，按照SOP，依次部署林根域DC(`ROOT-DC01`)和`chunming`子域DC(`CM-DC01`)。 | 无 | [[C.02-ActiveDirectory-部署SOP (多域森林版).md]] |
| **5** | **部署新DHCP服务器** | 在ESXi-B上创建**新VM**，安装DHCP角色并在`chunming`域中授权。创建所有作用域，**但保持禁用状态**。 | 无 | [[C.06-DHCP服务配置-SOP.md]] |
| **6** | **重装服务器A** | 将A站点的服务器重装为ESXi。 | A站点服务器 | - |

## 3. 第二阶段：实施期 (新设备到货后)

- **目标**: 执行核心网络割接，启用新服务，并完成用户迁移。**本阶段操作具有高风险，必须在计划的停机维护窗口内进行**。

| 步骤 | 任务描述 | 影响范围 | 实施指南 (SOP) |
| :--: | :--- | :--- | :--- |
| **7** | **部署新防火墙与AC** | **(关键)** 将新防火墙和AC设备替换掉旧的出口路由器。在新防火墙上创建所有VLAN的三层网关接口。 | **全网中断** | [[H.01-防火墙基础策略-SOP (深信服NGAF).md]], [[H.05-上网行为管理部署-SOP.md]] |
| **8** | **清理临时网关** | **(关键)** 在新防火墙接管网关功能后，立即**删除**在第一阶段创建于核心交换机上的所有**临时三层网关接口**。 | 核心网络 | - |
| **9** | **执行DHCP切换** | **(关键)** **(1)** 在新防火墙上配置所有VLAN的DHCP中继。 **(2)** 在新的Windows DHCP服务器上**启用**所有作用域。 | 全网 | [[C.06-DHCP服务配置-SOP.md]] |
| **10**| **建立站点间VPN** | 在两端防火墙上配置IPsec VPN，打通A、B两个站点。 | 核心网络 | [[H.02-站点间IPsec-VPN配置-SOP (深信服NGAF).md]] |
| **11**| **部署A站点子域** | 在A站点的ESXi上，按照SOP，部署`huazhi`子域的第一台DC(`HZ-DC01`)。 | 核心服务 | [[C.02-ActiveDirectory-部署SOP (多域森林版).md]] |
| **12**| **迁移用户与服务**| 在新架构稳定后，分批将用户PC和现有服务加入各自对应的新域中。 | 分批进行 | - |
