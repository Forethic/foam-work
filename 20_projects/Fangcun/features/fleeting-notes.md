# 闪念笔记 (Fleeting Notes)

**目标：** 随时随地、快速、无打扰地捕捉一闪而过的灵感、想法、任务，并能方便地整理和回顾。

**场景描述：**

1. **场景：通勤路上捕捉灵感 (移动端)**
    - **用户：** 您在公交/地铁上，突然想到一个“方寸”App的新功能点：可以根据笔记内容自动推荐相关标签。
    - **交互：** 您解锁手机，无需打开App，通过桌面小部件或快捷操作，直接弹出一个简洁的“快速笔记”输入框。您迅速输入：“新功能：笔记内容自动推荐标签。”
    - **结果：** 笔记被快速保存，并自动同步到云端。您没有被App的复杂界面打扰，灵感被及时捕捉。

2. **场景：桌面工作时记录思考 (桌面端)**
    - **用户：** 您在电脑前进行深度工作，阅读一篇技术文章，对文章中的某个概念有了新的理解。
    - **交互：** 您不需要切换应用，通过一个全局快捷键 (例如 `Ctrl + Shift + N`) 呼出一个浮动的小窗口。窗口中是您熟悉的Markdown编辑器。您输入：“理解了xxx概念，可将其与ooo结合，形成新思路。”
    - **结果：** 笔记被保存，并以Markdown格式存储。您可以在其中使用 `#思考`, `[[文章链接]]` 等Markdown语法进行关联，它将自动被解析并存储到数据库中。

3. **场景：回顾与整理 (桌面端)**
    - **用户：** 每周五下午，您习惯性地回顾本周的闪念笔记，从中提炼有价值的信息。
    - **交互：** 您打开“方寸”桌面App，进入“闪念笔记”视图。笔记以时间流的形式展示。在侧边栏或顶部筛选区，您可以根据**时间线（例如：今天、本周、本月）、标签（#思考、#项目A）、关键词**进行快速筛选和搜索。您看到上周捕捉的“新功能：笔记内容自动推荐标签”这条笔记。
    - **结果：** 您可以点击这条笔记，在编辑器中对其进行深化、补充细节，甚至将其直接转化为一个“待办任务”或“项目需求”卡片。

## 优化建议：打通后续转化路径

为了让捕捉到的灵感能高效地进入后续工作流，减少手动操作，建议进行以下优化：

- **提供“一键转化”入口**:
  - 在每一条闪念笔记旁边，设置一个“转化”或“提升”按钮。
  - 点击后允许用户将该笔记**一键转化**为：
    - **一个日程任务**: 笔记内容自动成为任务标题。遵循“即时创建”原则，如果任务需归属到一个新项目（如通过 `#新项目` 语法），该项目会被自动创建，然后跳转到 `日程安排` 模块设定日期。
    - **一个知识点/卡片**: 将笔记存入一个永久的知识库或卡片盒中，方便后续引用和关联。
    - **一个项目/文档的素材**: 将笔记内容追加到某个指定的项目文档或会议纪要中。
- **增强语法识别**:
  - 扩展文本识别能力。当用户输入“明天下午3点提醒我开会”，系统不仅能识别时间，还能在保存时**自动建议**创建为 `日程安排` 中的任务，实现从捕捉到安排的无缝衔接。
