# 业务流程编码与命名规范

为确保企业业务架构（Business Architecture）的清晰性与可维护性，本项目采用统一的“领域-序号”编码体系对业务流程进行标识。

## 1. 编码结构

所有业务流程文件与图表均遵循以下命名格式：

**`[领域代码][序号]_[流程名称]`**

* **领域代码**：1-2位大写字母，代表业务职能领域（Domain）。
* **序号**：2位数字，代表该领域下的流程序列（01, 02, ...）。
* **流程名称**：简明扼要的中文描述。

例如：`P01_供应商准入与管理流程`

## 2. 领域代码定义表

### 核心价值链 (Core Value Chain)

| 领域代码 | 含义 | 英文全称 | 涵盖范围 |
| :--- | :--- | :--- | :--- |
| **M** | 生产/制造 | Manufacturing | 生产计划、工单执行、车间管理、设备维护 |
| **P** | 采购 | Procurement | 供应商管理、采购申请、订单执行、入库协同 |
| **S** | 销售 | Sales | 客户管理、商机、报价、合同、订单、发货 |
| **W** | 仓储/物流 | Warehouse | 出入库、盘点、调拨、库位管理 |
| **R** | 研发 | R&D | 产品设计、BOM管理、变更管理、实验室管理 |
| **Q** | 质量 | Quality | 来料检验、过程检验、成品检验、质量追溯 |

### 职能支撑域 (Support Functions)

| 领域代码 | 含义 | 英文全称 | 涵盖范围 |
| :--- | :--- | :--- | :--- |
| **F** | 财务 | Finance | 应收应付、总账、成本核算、预算管理、税务 |
| **H** | 人力资源 | HR | 招聘、入转调离、薪资、绩效、培训 |
| **A** | 行政 | Administration | 物资管理（非IT）、车辆、宿舍、食堂、安保 |
| **L** | 法务/合规 | Legal | 合同审核、知识产权、纠纷处理、印章管理 |
| **PM** | 项目管理 | Project Mgmt | 项目立项、WBS分解、进度监控、工时管理 |

### 数字化与技术域 (Digital & Technology)

| 领域代码 | 含义 | 英文全称 | 涵盖范围 |
| :--- | :--- | :--- | :--- |
| **I** | 信息技术 | IT / InfoSec | 硬件资产、网络安全、账号权限、桌面支持 |
| **D** | 数字化/研发 | Digital / Dev | 软件研发流程、需求管理、发布上线、数据治理 |
| **T** | 技术架构 | Technology Arch | 技术选型、架构评审、云资源管理 |

## 3. 命名示例

* **M04_设备维护与保养流程**：属于制造域（M），第04号流程。
* **W01_物料入库流程**：属于仓储域（W），第01号流程。
* **D01_软件研发与发布流程**：属于数字化域（D），第01号流程。
* **I01_IT资源与服务管理流程**：属于IT基础域（I），第01号流程。

## 4. 维护原则

1. **唯一性**：确保每个编码在全局范围内唯一。
2. **稳定性**：流程编码一旦定稿，尽量不更改；流程升级时保留原编码，仅更新版本号。
3. **扩展性**：各领域下的序号预留足够的增长空间。
