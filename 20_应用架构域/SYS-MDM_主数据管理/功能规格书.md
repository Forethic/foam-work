# SYS-MDM 主数据管理系统功能规格书

**建设优先级**：P0 (极高)

## 1. 系统概述

**主数据管理（MDM）**系统是企业数据的“单一事实来源（SSOT）”。它解决跨系统间核心数据编码不统一、属性不一致、标准不规范的问题。通过建立统一的数据标准和分发机制，MDM确保所有业务系统在使用同一套语言对话。

## 2. 关联业务流程

本系统直接支撑以下核心业务流程：

- **[BIZ-FLOW-C03 主数据管理流程](../../10_业务架构域/10.1_业务流详细设计/BIZ-FLOW-C03_主数据管理流程.md)** (核心流程：创建、清洗、分发)
- **[BIZ-FLOW-R01 研发立项到技术转移](../../10_业务架构域/10.1_业务流详细设计/BIZ-FLOW-R01_研发立项到技术转移.md)** (物料主数据源头)

## 3. 核心功能模块

### 3.1. 主数据建模 (Master Data Modeling)

- **多模型支持**: 支持物料、客户、供应商、BOM、会计科目、人员、组织机构等多种主数据模型的定义。
- **属性管理**: 支持定义文本、数字、日期、枚举、布尔等多种类型的属性，并支持属性分组（视图）。
- **关系定义**: 支持主数据实体间的关联关系定义（如：BOM父子关系、客户与销售区域关系）。
- **模型版本**: 支持模型定义的版本管理，适应业务变更。

### 3.2. 数据标准与质量 (Standards & Quality)

- **编码规则引擎**:
  - 支持"前缀+日期+流水号+属性值"等灵活组合的自动编码规则。
  - 支持断号补全和手动/自动编码切换。
- **数据校验规则**:
  - **完整性**: 必填字段检查。
  - **有效性**: 数据格式（如邮箱、手机号）、值域范围检查。
  - **唯一性**: 关键字段（如税号、身份证号）查重。
- **清洗与查重**:
  - 内置模糊匹配算法（Levenshtein距离等），并在创建时实时提示疑似重复记录。
  - 支持数据合并（Merge）和拆分（Split）操作，维护"Golden Record"。

### 3.3. 生命周期管理 (Lifecycle Management)

- **数据维护门户**:
  - 提供统一的Web端数据申请、查询、变更入口。
  - 支持Excel批量导入/导出与校验。
- **流程审批 (BPM Integration)**:
  - 针对不同数据类型配置不同的审批流（如：原材料需采购、质检、财务审批；产成品需研发、财务、销售审批）。
  - 支持加签、转签、驳回和撤回。
- **版本控制**:
  - 记录每一次变更的"快照"，支持数据回溯（Time Travel Query）。
  - 完整的审计日志（Audit Trail）：谁、在什么时间、修改了什么字段、从什么值变为什么值。

### 3.4. 分发与订阅 (Distribution)

- **分发策略**:
  - **推模式 (Push)**: 数据生效后自动推送到订阅系统。
  - **拉模式 (Pull)**: 下游系统按需调用API获取。
- **过滤与转换**:
  - **按系统订阅**: 支持按接收系统订阅不同的视图（如：CRM订阅销售视图，WMS订阅仓储视图）。
  - **字段映射**: 支持不同系统间的字段名与码值转换（Mapping）。

### 3.5. 数据治理驾驶舱 (Data Insight)

- **质量仪表盘**:
  - **完整性看板**: 监控关键字段填报率（例：物料毛重缺失率、客户行业分类缺失率）。
  - **准确性看板**: 展示潜在重复数据数量、校验规则失败记录数。
- **流程绩效 (SLA)**:
  - 统计主数据创建/变更请求的平均处理时长、按时完成率。
  - 识别流程瓶颈节点（如：哪个部门审批最慢）。

## 4. 关键数据实体详细设计

### 4.1. 物料主数据 (Material)

作为最复杂的主数据，建议采用多视图管理：

- **基本视图 (Basic Data)**: 物料编码、描述（多语言）、基本计量单位、物料组、毛重/净重/体积、条码。
- **采购/库存视图 (Purchasing/Storage)**: 采购单位、批次管理标识、保质期、ABC分类、循环盘点标识。
- **销售视图 (Sales)**: 销售单位、销售组织、分销渠道、税务分类、装运组。
- **财务/成本视图 (Finance/Costing)**: 评估类（决定会计科目）、价格控制（标准价/移动平均价）、标准成本、成本核算批量。
- **质量视图 (Quality)**: 检验类型（IQC/FQC）、是否需检验、检验标准关联。

### 4.2. 商业伙伴 (Business Partner)

统一管理客户和供应商，包含公共部分和角色特定视图：

- **公共信息**: BP编码、名称、统一社会信用代码（税号）、注册地址、联系方式、银行账户信息。
- **客户角色视图 (Customer)**: 销售地区、客户组、价格列表、付款条件、信用额度、对账科目。
- **供应商角色视图 (Vendor)**: 采购组织、货币、付款条款、贸易条款 (Incoterms)、基于GR的IV验证标识、对账科目。

### 4.3. 财务主数据

- **会计科目 (G/L Account)**: 科目表、科目编码、科目名称、科目类型（资产/负债/损益）、是否未清项管理。
- **成本中心 (Cost Center)**: 编码、名称、负责人、层级结构。
- **利润中心 (Profit Center)**: 编码、名称、关联的成本中心组。

## 5. 业务规则与数据标准

### 5.1. 编码规则示例

- **原材料**: `10` + `大类(2位)` + `中类(2位)` + `流水号(4位)` (例: 1001010001)
- **半成品**: `20` + `流水号(6位)`
- **产成品**: `30` + `产品线(2位)` + `流水号(5位)`
- **客户**: `C` + `年份(2位)` + `流水号(5位)` (例: C2600001)
- **供应商**: `V` + `年份(2位)` + `流水号(5位)` (例: V2600001)

### 5.2. 关键校验规则

- **物料**: 同一图号/规格型号的物料不允许重复创建。
- **客户/供应商**: 统一社会信用代码（Tax ID）必须全局唯一，若重复则提示引用已有记录。
- **银行账户**: 格式必须符合IBAN或当地银行标准。

## 6. 系统集成设计

MDM 作为企业数据的交换枢纽，遵循 **"Global Standard, Local Execution"** 的原则。

### 6.1. 上游数据源 (Upstream Sources)

MDM 并不总是产生数据的地方，部分主数据来源于专门的业务系统：

- **PLM (研发)**: 推送**工程物料**、图号、设计属性到 MDM。MDM 在此基础上补充财务、采购、销售属性，转化为"制造物料"。
- **SRM (采购)**: 推送**潜在供应商**信息到 MDM。在供应商通过准入审核后，MDM 生成正式供应商编码并分发。
- **CRM (销售)**: 推送**潜在客户** (Prospect) 信息。在签订合同时，MDM 清洗并生成正式客户编码。

### 6.2. 下游消费端 (Downstream Consumers)

- **ERP (核心交易)**: 接收完整的物料、客商、财务主数据，确保订单、发票、财务凭证的基础数据准确。
- **WMS (仓储)**: 接收物料主数据（重点关注：长宽高、重量、保质期管理标识、批次标识）。
- **MES (制造)**: 接收生产物料清单与工艺相关的物料属性。
- **BI (分析)**: 接收所有维度的标准主数据，作为企业级数据仓库的一致性维度（Conformed Dimensions）。

## 6. 系统集成设计

### 6.1. 集成架构图示

MDM作为Hub，采用**Publish-Subscribe**模式。

### 6.2. 数据流向定义

| 数据对象 | 源头系统 | 触发条件 | MDM处理 | 目标系统 (订阅方) | 集成方式 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| **物料 (研发)** | PLM | 研发设计发布/归档 | 解析、校验、生成正式编码、补充财务/采购视图 | ERP(FIN/SCM), MES, WMS, SRM, CRM | MQ (异步) |
| **物料 (包材/辅料)** | SRM/Email | 采购员在MDM手动申请 | 审批流、查重、编码 | ERP, WMS, SRM | MQ (异步) |
| **客户 (正式)** | CRM | 销售线索转正式客户 | 接收申请、财务审核信用、分配编码 | ERP(FIN/SD), CRM, TMS, WMS | MQ (异步) |
| **供应商** | SRM | 潜在供应商准入审核通过 | 接收申请、风控审核、分配编码 | ERP(FIN/MM), SRM, QMS | MQ (异步) |
| **人员/组织** | HR | 入职/离职/组织变更 | 接收同步 | OA, ERP, CRM, MES (全员同步) | API (定时/实时) |
| **会计科目** | ERP(FIN) | 财务主管在MDM手动维护 | 审批通过 | ERP, SRM, CRM (作为参照) | API (定时) |

### 6.3. 接口标准

- **协议**: RESTful API for CRUD; AMQP (RabbitMQ) for distribution.
- **幂等性**: 所有分发消息带有唯一的`message_id`和`version_id`，下游系统需处理乱序与重复消费。
